# Review 5 — UI / 视觉设计评审 (RummyCube)

_独立视觉评审，聚焦布局、层级、排版与打磨。素材：`review5-01-landing`、`review5-02-board-coach`、`review5-03-board-hints` + `board.css` / `classic.css` / `lobby.css` / `chat.css`。仅分析，未改任何源码。_

## TL;DR
当前对局画面读起来是「**一大片空旷绿毡 + 右侧一根半空的死栏 + 被挤在底部的木架条**」——纵向与横向都失衡。**最高价值的单点修复：回收聊天栏永久预留的 316px 右侧 gutter（`board.css:33`），并把空旷的 9×32 棋盘托盘在开局阶段收敛/填充空状态**，让三个区域（board / rack / chat）重新配平。这一改动同时缓解「上空下挤」和「右侧死栏」两大观感问题。次要但低成本的高回报项在落地页：默认就灰掉的 Create 按钮和一排 `0` 统计在劝退转化。

---

## Findings(按价值排序)

### 1. 聊天栏永久吃掉 316px 右栏，且大量留白 — Impact: High × Effort: M
**问题**：`.board-container` 在 `>=821px` 无条件 `padding-right: calc(300px + 16px)`（`board.css:33`），而聊天面板本身只有 `width:300px; max-height:42vh`（`chat.css:14,16`）。结果如 `review5-03`：聊天框只占右上一小块，**它下方约 60% 的高度是纯绿死区**，棋盘却被这条 316px 的 gutter 永久挤窄。单人/对 bot 时聊天价值最低，却占据了最强的视觉权重（固定、贴右上、亮棕标题栏）。
**为什么重要**：这是「空旷感」的最大放大器——既浪费横向空间，又让右侧显得未完成。
**建议**：
- 桌面默认把聊天**折叠为右下角 FAB**（移动端已有 `.chat-fab` 逻辑，复用即可），有未读再气泡提醒；展开时浮在棋盘之上而非预留 gutter。
- 若坚持常驻，gutter 收窄到 `260px` 并让面板**贴底/限高**，把回收出的宽度还给 `.ref` 棋盘托盘。
- 至少：聊天面板加 `min-height` 不变、但 gutter 用 `flex` 让棋盘吃满剩余宽度，消除右下死区。

### 2. 纵向失衡:空棋盘托盘 vs 挤压的底部三分之一 — Impact: High × Effort: M
**问题**：棋盘网格固定 9 行 × 32 列、cell `2.2vw × 7vh`（`constants.js:4-8`、`GridContainer.jsx:19,41`）≈ **63vh** 的托盘，开局几乎全空（`review5-02/03` 的大片网格线）；而 avatar、turn-banner、木架、控制行全部塞进底部约 40%。`.board` 顶部留白 `2.5vh`（`board.css:50`）+ 空托盘把重心强行往下压。
**为什么重要**：视觉重心落在底部一条，上方 2/3 是「等待被填充的空」，缺乏高级感与稳定的画面结构。
**建议**：开局/空盘时让网格托盘**纵向收敛**（如 `max-height: 46vh` 居中，落子后再增长），把 `.hand-buttons` 上移（`margin-top` 由 `2.2vh` 提到 `3.2vh` 区间），让 rack 与控制行落在更接近视觉中线的位置；空与满两态之间用过渡平滑增高。

### 3. 落地页主 CTA 默认就是「灰禁用」态 — Impact: High × Effort: S
**问题**：`review5-01` 里整页英雄区下方最重要的 **Create 按钮呈灰棕禁用色**——因为用户名为空触发 `.lobby-btn:disabled`（`lobby.css:169-175`，渐变 `#ddd2b4→#cabd98`、文字 50% 透明）。页面的转化主按钮一上来就是「死」的。
**为什么重要**：首屏主行动号召看着不可点 = 直接掉转化，且与绿色品牌主按钮（`.lobby-btn-primary` 绿 `#3a7d50→#205233`）形成「我坏了」的错觉。
**建议**：默认保留**绿色 active 皮肤**，点击空用户名时再就地提示（输入框抖动/红描边/「先取个名字」），而不是用全灰禁用态做首屏默认；或把焦点自动落在 Username 输入框，引导填写。

### 4. 一排 `0` 统计 = 负向社会证明 — Impact: Med × Effort: S
**问题**：`review5-01` 卡片下方 `0 games in progress / 0 rooms waiting / 0 players online`（`.server-stats`，大号 brass 数字 `1.7rem`，`lobby.css:347-352`）被显眼地展示。
**为什么重要**：三个醒目的 `0` 在向新访客喊「这游戏没人玩」，劝退效果强，且占了仅次于卡片的视觉权重。
**建议**：统计为 0 时**隐藏整块**或替换为友好文案（「Be the first to start a game 🎴」）；仅当任一指标 `>0` 才以当前样式展示，把「热度」当正向信号、把冷启动藏起来。

### 5. 空棋盘没有空状态引导,读作「空洞」 — Impact: High × Effort: M
**问题**：空托盘只有 4% 白的网格线（`board.css:300-303`）——是一片无意义的网格虚空。唯一引导是漂在木架上方的小 hint pill「No tiles fit the table yet」（`Board.jsx:624`），与空棋盘脱节。
**为什么重要**：开局是新玩家第一印象，最大的画面区域却没传达「在这里拼牌」。
**建议**：在 `.ref` 托盘**居中**放一个低透明度空状态（图标 + 文案，如「把牌拖到这里组成第一组 ≥30 分的 run/set」），落子后淡出。让虚空变成有目的的「桌面」。配合 #2 的托盘收敛，空状态盒会更聚焦。

### 6. avatar + 计时环 + Your turn 横幅悬在毡/架接缝上,像漂浮的剪贴物 — Impact: Med × Effort: M
**问题**：`.rack-self` 绝对定位、贴在木架左上外侧（`bottom: calc(100% + 2px)`，`board.css:109-114`），80px 头像外套 100px 计时环 SVG（溢出头像盒）、右下又挂红色 tile-count 角标，左上再浮一条 `Your turn` 胶囊（`left:92px; bottom:calc(100%+26px)`，`board.css:581-582`）。如 `review5-02/03`：整组横跨绿毡与木架的缝、无承载容器，读作「贴上去的浮层」而非桌面的一部分。
**为什么重要**：界面最该锚定的元素（你、你的回合、剩余时间）却最缺结构感，削弱「真实牌桌」的高级感。
**建议**：给自己头像做一个**小木质名牌底座**（与 `--rack` 同材质），把 avatar + 名字 + 回合点稳稳坐进木架左上角；turn-banner 改为名牌内的一行/一个点，去掉额外漂浮胶囊。让这组成为木架的延伸，而非毡上的孤岛。

### 7. 计时环用纯 RGB 蓝→红,跳出整套 brass/绿 调性 — Impact: Med × Effort: S
**问题**：环色由 `rgb(redIntensity,0,blueIntensity)` 实时插值，起点纯蓝 `#00f`（`PlayerAvatar.jsx:15,30`）。纯蓝/纯红在这套低饱和的绿毡 + 黄铜 + 象牙里非常刺眼、廉价。
**为什么重要**：最显眼的动态元素却是全场唯一的高饱和原色，破坏品牌色一致性。
**建议**：把进度色映射到**品牌色坡道**——充足时 brass `#cda24b`/绿，过半转琥珀 `#e0a64b`，最后 5s 才转告警红 `#b3162a`（与 `--c-red` 对齐）。已有 `.timer-low` 钩子可承接末段状态。

### 8. 排版混用 Segoe UI 与品牌 Poppins — Impact: Med × Effort: S
**问题**：品牌字是 Poppins（`index.html:16` 引入；wordmark、tiles、primary/secondary-action 均用），但 `.turn-banner`、`.timer-seconds`、基类 `.rummikub-button` 仍是 `'Segoe UI'`（`board.css:591,563,608`）。同屏两套无衬线，字形/字重观感不统一。
**为什么重要**：排版是「高级感」最敏感的维度，混排会被潜意识感知为不精致。
**建议**：把上述三处统一为 `'Poppins', system-ui, sans-serif`；计时数字若想更几何可单列一个 tabular/等宽数字栈，但不要回退到 Segoe UI。

### 9. wordmark 两处不同色 + 英雄标题在绿底对比偏低 — Impact: Med × Effort: S
**问题**：同一个「RummyCube」字标，导航栏是 **cream `#f5edd8`**（`index.css:56`），落地英雄区却是 **brass `#cda24b`**（`lobby.css:36`）。且 brass 标题压在中绿毡上，对比仅 ~2.2:1，低于大字 3:1 的 AA 线（`review5-01` 标题略「发糊」）。
**为什么重要**：字标是品牌锚点，两种颜色削弱识别度；英雄标题对比不足影响可读性与气场。
**建议**：定一个**唯一的 wordmark 处理**（建议 cream 字 + 细 brass 描边/底纹，两处一致）；英雄标题加深色 `text-shadow`/描边把对比拉到 ≥3:1，或字色提亮到 `#e7c97a` 同时压暗其下毡色。

### 10. 木架右侧大片空棕,牌左对齐显得未完成 — Impact: Med × Effort: S
**问题**：`.hand-buttons` 为 `width: fit-content`（`board.css:98`），宽度被下方更宽的控制行撑开（`controls-wrapper` 三列，`board.css:179-185`）；14–15 张牌左对齐后，右侧留下一大片空木纹（`review5-02/03` 牌只到约 565px，木架延伸到约 845px）。
**为什么重要**：架子看着「半空」，和底部拥挤感矛盾，整体不平衡。
**建议**：把牌组在木架内**居中**（hand grid 容器 `justify-content:center` 或 `margin:0 auto`），或让木架宽度**收敛到牌组宽度**、控制行单独成行；让空木纹左右对称，架子读作「刚好装满」。

### 11. Undo/Redo 幽灵按钮,贴在毡/架缝上几乎隐形 — Impact: Low × Effort: S
**问题**：`.icon-button` 背景 `rgba(20,16,10,.42)`（`board.css:139`），`.rack-tools` 浮在木架右上外侧的绿毡上（`board.css:119-123`）。`review5-02/03` 里两个 ↶/↷ 几乎看不见，且压在毡上像被禁用。
**为什么重要**：可逆操作能降低试错焦虑，但当前可发现性接近于零。
**建议**：把底色提到 `rgba(40,30,16,.6)` + 1px brass 描边更实，或像建议 #6 一样**坐进木架内**（与 `.rack-self` 对称）形成稳定承载，而非浮在毡上。保持「安静」但要「可见」。

### 12. 圆角/阴影体系零散,缺统一刻度 — Impact: Low × Effort: M
**问题**：圆角横跨 tile `5px`(`:359`)、grid-item `6px`(`:313`)、icon/secondary `10px`(`:137,755`)、primary/rummikub `12px`(`:702,614`)、chat-panel `14px`(`chat.css:20`)、rack `16px`(`:102`)、lobby-card `18px`(`lobby.css:80`)——七档无规律；阴影也各写各的偏移。
**为什么重要**：缺少 radius/elevation token 会让组件之间「不像一套」，是高级感的隐性扣分项。
**建议**：定义 4 档 token（如 `--r-sm:6 / --r-md:10 / --r-lg:14 / --r-xl:18`）与 2–3 档 elevation 阴影，全量替换硬编码值；保留 `999px` 仅作 pill。低优先，建议作为一次集中收敛。
