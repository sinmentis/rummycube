## 1. 背景与现状(代码勘探所得)

owner 反馈 4 个 UX/bug + 2 个交互玩法。逐项核实当前真实实现:

1. **缺退出入口** —— solo 测试局与结算界面都无法退出。离开 = `navigate('/')`(`Match.jsx` 用 react-router);座位凭证存在 `localStorage['rummycube:match:<id>']`,可重连。
2. **「Your turn」压头像** —— `.avatar` 实为 **80px**(`board.css:437-440`);`.turn-banner` 定位 `left:6px; bottom:calc(100%+56px)`(`board.css:559-562`)正好落在头像顶部 → 重叠。
3. **聊天框遮桌面** —— `.chat-root` 是 `position:fixed; top:…; right:8px; z-index:65; width:300px`(`chat.css:2-15`),浮在右上、**未给棋盘留空位**,牌会滑到它底下。
4. **Give up 用浏览器弹窗** —— `onForfeitTurn` 用 `window.confirm`(`Board.jsx:320-325`)。
5. **Feature 1 长按拿整组** —— 代码里**已有一套没接线的** `handleLongPress`(`boardUtil.js:114-149`,只向右走且按 `isSequenceValid` 停)+ 现成的多选/多拖(`state.selectedTiles` 驱动 `onDragStart`/`DragOverlay`/`onDragEnd`→`moveTiles`)。`GridContainer` 没把 `handleLongPress` 透传、`Tile.jsx` 无长按事件 → 长按当前是死代码。
6. **Feature 2 自动吸附+挤开** —— `resolveDropSlot`(`dndUtil.js:62-104`,纯函数)只在**同一行的空位**落子、**从不位移已有牌**,放不下就 `{ok:false}` 拒绝;`moveTiles` 的 `insertTile`/`isOverlap`(`moves.js:61-115`)碰撞即 `INVALID_MOVE`。**没有任何重排/挤位 helper**。行模型 = 单行最多 `BOARD_COLS=32` 个独立列、允许空格;一组 = 连续占位(`extractSeqs` 按行分组、列连续即同组、空一格即断组,`moveValidation.js:25-66`)。

## 2. 设计目标

- 补齐基本可用性(退出、反确认、不遮挡)。
- 两个交互玩法降低操作负担:长按整组拿起、拖放自动吸附并挤位排列。
- **不改 Rummikub 规则/计分/反作弊/合法性校验**;摆放仍在提交时由 `isBoardValid` 裁定。挤位是纯几何排版,不限制数字。
- 服务器权威:所有改 `G.tilePositions` 的操作都走服务端 move(含新的挤位 move),客户端只算目标、不改权威态。

## 3. 工作项

### WS-1 退出按钮(`go_home_rejoinable`)

- **目标:** 任意时刻可退出回主页;对局保留可重连;结算界面也能回主页。
- **改动:**
  - 顶栏(How to play / Mute 一带)加一个 `Exit` 按钮 → `navigate('/')`。**非破坏性、单击**(座位凭证保留 → 可从主页重进)。
  - `GameOverModal` 加一个 `Back to home` 按钮 → `navigate('/')`(结算后清掉 `rummycube:match:<id>` 避免再自动重连)。
- **涉及:** `Board.jsx`(顶栏渲染 + `useNavigate`)、`GameOverModal.jsx`、`board.css`/`GameOverModal.css`。
- **边界:** 多人局退出 = 仅离开本端 socket(对局/座位仍在,靠现有 grace/重连);不发任何服务端「离开」move。
- **测试:** Exit 按钮存在且 `onClick` 触发 navigate('/')(RTL,mock router);GameOver 的 Back-to-home 同理 + 清 localStorage 键。

### WS-2 Give up 两次点击确认(`two_click`)

- **目标:** 去掉浏览器 `window.confirm`,改成同位置两次点击确认。
- **改动:** `Give up turn` 按钮加本地「armed」态:第一次点 → 文案变 `Click again to confirm` + 警示色;起 `GIVEUP_CONFIRM_MS`(默认 3000)[PLACEHOLDER] 计时,未再点则复原;**armed 期间**第二次点(同一按钮)→ `moves.forfeitTurn()` + 复原。组件卸载/回合切换清计时器与 armed 态。
- **涉及:** `Board.jsx`(`onForfeitTurn` 改为本地状态机,移除 `window.confirm`)、`board.css`(armed 警示样式)。
- **边界:** 切到别人回合 / gameover / 提交成功 → 立即复位 armed;armed 文案英文;动画(若有)受 reduced-motion 门控。
- **测试:** 第一次点不 forfeit、文案/类切换;armed 内第二次点触发 `forfeitTurn`;`advanceTimersByTime(3000)` 后复原且不 forfeit(RTL,fake timers)。

### WS-3 聊天 dock 留空位(`dock_gutter`)

- **目标:** 桌面端棋盘内容/横向滚动永不滑到聊天框底下。
- **改动:** 桌面(聊天常驻的宽屏)给棋盘容器右侧预留 = 聊天宽度(300px)+ 间距的**留白槽**(右 padding/margin 或 `max-width`),使可玩区止于聊天左缘;聊天仍 `fixed`。窄屏沿用现有 FAB 折叠,不变。
- **涉及:** `board.css`(棋盘容器在 `min-width` 断点上的右留白)、必要时 `chat.css`。需确认棋盘横向滚动区(`.board`/`.board-container`,`overflow-x`)宽度计入留白。
- **边界:** 断点要与聊天 FAB 的折叠断点一致(避免「既折叠又留白」或「既常驻又遮挡」的中间带);留白只在聊天常驻时生效。
- **测试:** 主要是视觉/布局(jsdom 测不了像素)——以 CSS 断点推理为准 + 部署后真机核验;可加一条「宽屏下棋盘容器带留白类」的 DOM 断言。

### WS-4 「Your turn」横幅移到头像右侧(`right_of_avatar`)

- **目标:** 横幅不再压住头像/计时环。
- **改动:** `.turn-banner` 由 `left:6px; bottom:calc(100%+56px)`(压 80px 头像)改为头像**右侧**:`left: ~92px`(头像 left:6px + 80px + ~6px 间距),垂直对齐头像中部(如 `bottom: calc(100% + 26px)`),读作「活动头像旁的标签」。窄屏断点(`board.css:904+`)按需微调,保证不溢出。
- **涉及:** `board.css`(`.turn-banner` 定位;移动端块)。
- **边界:** 横幅 `white-space:nowrap`,长名「{name}'s turn」在窄屏不溢出/不再压其它元素;`z-index` 维持不挡牌。
- **测试:** 视觉为主(同 WS-3,CSS 推理 + 真机);保留 `turnBannerLabel` 既有单测。

### WS-5 Feature 1 — 长按拿整组

- **目标:** 在**手牌或桌面**长按一组里的任意一张,整组(同行连续占位的 run/set)一次性拿起拖动,免去逐张点选。
- **设计:**
  - 新增纯函数 `contiguousGroup(tilePositions, pressedTileId) -> tileId[]`:取按下牌的 `{gridId,row,col}`,向左(col-1…)和向右(col+1…)在**同 grid、同 row**(手牌还需同 `playerID`)连续占位处扩展,返回含按下牌在内的整段连续 run。
  - **激活:** 在牌上按住 ~`LONG_PRESS_MS`(默认 200)[PLACEHOLDER];计时到点(期间未抬起/未超过拖拽位移阈值)→ `setState.selectedTiles = contiguousGroup(...)`,随后现有 @dnd-kit 拖拽(`distance:6`)带动整组(`DragOverlay` 已渲染整组、`onDragEnd`→`moveTiles` 落整段)。**短按(tap)**仍走现有单选/toggle(`Tile.jsx` 的 `onClick`/`handleTileSelection`)。
  - 复用现有「selectedTiles → 多拖 → moveTiles」管线;把死掉的 `handleLongPress` 接线(经 `GridContainer`/`Tile`)或以新的 press-timer 实现并替换它。
- **涉及:** `Tile.jsx`(press-hold 事件,和 `touchAction:none`+dnd listeners 共存)、`GridContainer.jsx`(透传)、`Board.jsx`(回调)、新 `contiguousGroup`(`boardUtil.js` 或新文件)。
- **边界:** 单张(无邻居)长按 = 单张;手牌组按 `playerID` 隔离;长按与拖拽/滚动手势不打架(hold 不动=选组,动=拖);桌面已提交牌与 tmp 牌都可成组拿起(规则同 `fromBoardToBoard`)。
- **测试:** `contiguousGroup` 纯单测(左右扩展、断组于空格、手牌跨 playerID 不串、单张退化);长按填充 `selectedTiles` 的交互(RTL + fake timers,断言 hold 后 selection = 整组、tap 仍单选)。

### WS-6 Feature 2 — 拖放自动吸附 + 插入挤位(完整版,含级联 + 双向)

- **目标:** 把 N 张牌拖到桌面目标列 T 时:目标空位 → 直接吸附落子;若与已有牌冲突 → 自动在 T 处插入并把被顶的连续段**右移腾位**(级联直到遇到空格吸收),右侧到边则改**左移**,两侧都放不下才拒绝。例:`1 2 3 _ 7 7 7` 放 `4 5` 到 3 后 → `1 2 3 4 5 7 7 7`(三个 7 整体右移 1)。
- **核心纯函数(最高风险,务必充分单测 + rubber-duck):**
  `insertWithPush(rowTiles, T, N, maxCol) -> {shifts: {tileId: newCol}, newCols: number[]} | null`,`rowTiles` = 该行现有 `[{tileId, col}]`(不含拖入牌),`maxCol = BOARD_COLS-1 = 31`。
  - **目标全空**(`[T, T+N-1]` 均无占位)→ `shifts={}`,`newCols=[T..T+N-1]`(纯吸附,等同现状)。
  - **右向 ripple(主):**
    ```
    cursor = T + N
    for {tileId, col} in rowTiles where col >= T, 按 col 升序:
        if col < cursor:                 // 与新牌或前一张被推牌冲突
            if cursor > maxCol: 右向失败
            shifts[tileId] = cursor; cursor += 1
        else:
            break                        // 右侧有空格吸收,停止级联
    若未失败 → 返回 {shifts, newCols:[T..T+N-1]}
    ```
  - **左向 ripple(右向失败时的镜像回退):**
    ```
    cursor = T - 1
    for {tileId, col} in rowTiles where col <= T+N-1, 按 col 降序:
        if col > cursor:                 // 与新牌或前一张被推牌冲突
            if cursor < 0: 左向失败
            shifts[tileId] = cursor; cursor -= 1
        else:
            break                        // 左侧有空格吸收,停止级联
    若未失败 → 返回 {shifts, newCols:[T..T+N-1]}
    ```
  - 两向都失败 → 返回 `null`(拒绝,`buzz`,不落子,不改 G)。
- **新服务端 move `insertTilesWithPush`**(权威、原子):接收目标 `{gridId:'b', col:T, row}` + `selectedTiles`(已排序的 N 张),调 `insertWithPush`;为 `null` 则 `INVALID_MOVE`;否则在**一次 move 内**写入被推牌的新 col + 新牌落于 `newCols`(hand→board 置 `tmp:true`),单次 `gameStateStack` 入栈(一次撤销还原整组挤位)。仅 board 行(`gridId==='b'`)启用挤位;手牌行沿用现有 `moveTiles`(不挤位)。
- **客户端接线:** `onDragEnd`/`onCellTap` 在 board 落点先试现有 `resolveDropSlot`(全空即原路);命中占位则改调 `insertTilesWithPush`(目标 col = 指针所在列 T)。保持「失败=非破坏性拒绝」的现状 UX。
- **涉及:** 新 `insertWithPush`(纯,新文件如 `src/rummikub/insertPush.js`)、`moves.js`(`insertTilesWithPush` + 注册)、`Game.js`(注册 move)、`Board.jsx`(落点分流)、`dndUtil.js`(必要的占位/目标计算)。
- **边界与失败态(必须测):** 右溢出(`>31`)→ 转左;左溢出(`<0`)→ 拒绝;级联跨多组(ripple 自然处理,遇空格即停);单张插入(N=1)同样挤位;被推牌含已提交 board 牌(同回合可重排)与 tmp 牌;挤位不得跨行;一次 move 原子完成(中途不触发 `isOverlap` 误判);撤销/重做按单步还原;反作弊不退化(只当前玩家、当前回合可发该 move)。**几何挤位不校验数字合法性**(合法性仍在提交时裁定)。
- **测试:** `insertWithPush` 纯单测覆盖:全空吸附、单层右推、级联右推(`1 2 3 _ 7 7 7` + `4 5`)、右溢出转左、双向都满拒绝(null)、N=1 插入、空格吸收停止级联、最大列边界。reducer 级:`insertTilesWithPush` 写入正确、原子、`INVALID_MOVE` 路径不改 G、撤销还原、anti-cheat。

## 4. 必须保持的不变量

- **服务器权威/反作弊不退化:** 所有摆放/挤位经服务端 move;新 move 仅当前玩家、当前回合可发;非法 → `INVALID_MOVE` 且不改 G(immer 草稿丢弃)。
- **合法性裁定时机不变:** 摆放/挤位是几何排版;run/set 合法性仍在 `submitMeld`/`isBoardValid` 提交时裁定。
- **门控/无障碍:** 新动画(armed 警示、长按反馈等)受 `prefers-reduced-motion` + mute 门控;`jsdom` 无 `matchMedia` → JS 动画判定带 `window.matchMedia &&` 守卫。
- **应用内文案英文**(spec/plan 中文)。
- **playerView 不泄露:** 挤位只在 board(公共)行;不暴露对手手牌。
- **无新依赖;`node src/server.js` 可启动**(新组件经 Vite 进 Board.jsx;服务端路径显式 `.js`);构建产物无新增 `console.log`。
- **既有能力不回归:** 现有多选/多拖、tap-to-place、键盘 Undo/Redo、断线/同步、各持久化键、超时公告、Hints 开关、主/次按钮、图标 Undo/Redo 全部保留。

## 5. 可调参数([PLACEHOLDER])

| 参数 | 默认 | 说明 |
|---|---|---|
| `GIVEUP_CONFIRM_MS` | 3000 | 两次点击确认的复位窗口 |

## 6. 不在本轮范围

- 不改 Rummikub 规则、计分、joker、首出 30 分门槛。
- 不引入新依赖、不做断线机制本身改动。
- F2 不做「拖动中实时预览挤位」(仅落子时计算);不做跨行挤位;不做「挤位时自动校验并阻止形成非法组」(几何排版,合法性提交时裁定)。
