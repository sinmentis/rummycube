## 1. 背景与现状

owner 选择「除 VM 多站点架构外的全部 backlog」。逐项核实现状:

- **combo「ON FIRE」几乎永远亮**:`comboMath.js` 的 `manipulationScore = 3*groups + 2*rearranged + 1*placed`,但 `comboLabel` 阈值才 3/5/7。一次 4 张牌或 2 组的普通出牌就 ≥7 → ON FIRE;甚至一次性堆 7 张牌(1 组)= 7 也 ON FIRE。分级失去意义。
- **retrieveJoker 偏严且未接 UI**:`moves.js:367` `JOKER_RETRIEVE_TILES_NEEDED=2`(手里要 2 张所代表的牌,但只用 1 张换上桌、第 2 张只是门槛),非标准、少见;且该 move(`moves.js:377`)还没有任何客户端入口。
- **moveTiles 既有瑕疵**:`insertTile`(`moves.js:84-115`)返回 `INVALID_MOVE` 时,`moveTiles` 的循环**不传播**该返回 → 拒绝的插入仍「成功」并已 push 了一条 undo 快照(幽灵 undo);且 `insertTile` 对 `currPos.gridId` 解引用无空值守卫。
- **DRY**:T4 的挤位分流分支在 `onDragEnd`/`onCellTap` 各抄一遍;`insertTilesWithPush`(`moves.js`)内联重排了 `boardRowTiles` 已做的事;`boardRowTiles` 硬编码 `'b'` 而非 `BOARD_GRID_ID`。
- **PlayerAvatar 掉线指示**仍读 metadata 的 `isConnected`,而非权威 `G.connected[seat]`(WS-12 已建立)。
- **视觉/无障碍遗留**:`.primary-action`/`.secondary-action` 没有 `.icon-button` 的可见焦点环;`.rack-tools` 断点 560 vs 控件 820、无 overlay 碰撞回退;`.timeout-toast` `nowrap` 在极窄屏可能截断;聊天留白槽露 `--felt-base` 缝。
- **tap-to-place 键盘路径**(WS-14 顺延)、**首次开启 Hints tooltip**(UX-pass T4 跳过)未做。

## 2. 设计目标

- 让 combo 反馈**分级有意义**(ON FIRE = 真正的大操作,而非堆牌)。
- 把 retrieveJoker 改回**经典 1 张**并**接上拖拽 UI**(把代表牌拖到桌上 joker 上换回)。
- 清掉既有正确性瑕疵(moveTiles 传播 INVALID_MOVE)+ DRY + 权威掉线指示 + 视觉打磨 + 两个顺延项。
- **不改 Rummikub 规则**(除 joker 取回张数,owner 决策)、不改服务器权威、合法性仍提交时裁定、应用内英文、无新依赖。

## 3. 工作项

### WS-A combo 调参(earned · 让分级有意义)

- **目标:** ON FIRE 只给真正的大操作(多组形成 + 棋盘重排),普通出牌给 NICE/COMBO,纯堆牌不再 ON FIRE。
- **改动(交游戏设计专家定最终曲线,均 [PLACEHOLDER] / playtest-tunable):**
  1. **重平衡 `manipulationScore`**:降低「纯张数」的支配力——把 `placed` 项**封顶**(如 `min(placed, PLACED_CAP)`)或降 `W_PLACE`,使分数主要由 `groups`(形成/延伸的合法组)与 `rearranged`(重排的既有牌)驱动。
  2. **重设 `comboLabel` 阈值**(`comboMath.js:14-19`),与新分数量纲匹配,使 NICE=扎实出牌、COMBO=多组/带重排、ON FIRE=明显的大操作。
  3. 不改 combo 的**触发管线**(`applyValidMove` 算 `G.lastPlay.combo`、`ComboOverlay` 渲染、`particleCount`/intensity 门控)——只调 `comboMath.js` 的纯函数;overlay 文案与粒子强度随阈值自然变温和。
- **涉及:** `src/rummikub/juice/comboMath.js`(纯)。可能 `juice/comboMath` 的消费点只读不改。
- **测试:** 纯单测——典型出牌矩阵(1 张接龙、3 张接龙、4-5 张多组、堆 7 张单组、含重排的大操作)→ 断言落入预期分级;堆牌不再 ON FIRE。

### WS-B retrieveJoker = 经典 1 张 + 拖拽 UI

- **目标:** 把桌上 joker 用它**所代表的那一张手牌**换回手里;交互 = 把该手牌**拖到桌上的 joker 上**。
- **服务端改动(`moves.js`):**
  - `JOKER_RETRIEVE_TILES_NEEDED = 1`;`retrieveJoker` 简化为 `retrieveJoker({G,ctx,playerID}, jokerTileId, tileId)`——单张:必须是当前玩家手牌、非 joker、`getTileValue===representedValue`、`deactivateTileVariant` 颜色由换后 `isBoardValid` 兜底。换:该手牌占 joker 的板位(tmp:false, playerID:null),joker 回到该手牌的手位;换后 `!isBoardValid` → 整体 no-op(`INVALID_MOVE`)。保留「仅当前玩家、joker 须在合法组、不结束回合/不抽牌」语义。
  - 仍是非破坏性:任何不合规 → `INVALID_MOVE`(immer 丢弃草稿)。
- **客户端拖拽分流(`Board.jsx` `onDragEnd`,在挤位分流**之前**):** 若 N=1、落点 board 格上是一张**已结算(非 tmp)的 joker**、且被拖手牌的值 == 该 joker 代表值 → 调 `moves.retrieveJoker(jokerTileId, draggedTileId)`(不走 place/push)。不匹配则继续既有分流(挤位/吸附)。给即时 `buzz` 反馈于客户端预判失败时(可选)。
- **涉及:** `moves.js`(常量 + move 签名简化 + 注册不变)、`Game.js`(已注册 retrieveJoker,签名变更需同步)、`Board.jsx`(onDragEnd 新增 joker-swap 分支)、可能 `dndUtil.js`(一个纯 helper:`jokerSwapTarget(tilePositions, cell, draggedTileId)` 判定)。
- **边界:** 拖非匹配牌到 joker → 不换,走既有逻辑;joker 不在合法组 / 拖的是 joker 本身 / 多选拖(N>1)→ 不触发取回;换后破坏棋盘 → no-op + buzz。**几何/合法性仍服务端裁定。**
- **测试:** 纯 helper `jokerSwapTarget` 单测;reducer 级 `retrieveJoker` 单张(成功换、值不匹配拒、非当前玩家拒、换后非法 no-op、joker 回手位);客户端分流 RTL(拖匹配牌到 joker → 调 retrieveJoker;拖不匹配 → 走 moveTiles/insertTilesWithPush)。

### WS-C moveTiles 加固(传播 INVALID_MOVE + 空值守卫)

- **改动:** `moveTiles`(`moves.js:77-128`)的 `insertTile` 调用**传播返回值**——任一 `insertTile` 返回 `INVALID_MOVE` 则整个 `moveTiles` 返回 `INVALID_MOVE`(immer 丢弃,含已 push 的 undo 快照一并回滚),不再「幽灵成功」。`insertTile` 开头加 `if (!currPos) return INVALID_MOVE`(`moves.js:89` 前)防解引用。
- **涉及:** `moves.js`。
- **边界:** 不改合法移动的行为;只把「本应拒绝」的路径真正拒绝。
- **测试:** reducer——一个会触发 `insertTile` 拒绝的 moveTiles → G 不变(含 `gameStateStack` 不增);既有 moveTiles 测试保持绿。

### WS-D DRY

- **改动:** 抽 `Board.jsx` `onDragEnd`/`onCellTap` 的挤位分流为一个共享 helper(纯判定 + 返回「该调哪个 move + 参数」或「buzz」);`insertTilesWithPush` 复用 `boardRowTiles`;`boardRowTiles` 用 `BOARD_GRID_ID` 常量。
- **涉及:** `Board.jsx`、`dndUtil.js`、`moves.js`。
- **测试:** 既有 T4/T3 测试覆盖;helper 抽取不改行为(回归绿即可)。

### WS-E PlayerAvatar → 权威 G.connected

- **改动:** 掉线徽标的数据源由 `matchData[seat].isConnected` 改为 `G.connected[seat]`(WS-12 服务端权威);`G.connected` 为空(WS-12 前的旧局)时回退 metadata。Board/TableSeats 把 `G.connected` 传给 `PlayerAvatar`。
- **涉及:** `PlayerAvatar.jsx`、`Board.jsx`/`TableSeats.jsx`。
- **测试:** RTL——`G.connected[seat]===false` → 显示掉线徽标;缺省回退 metadata。

### WS-F 视觉 / 无障碍打磨

- `.primary-action`/`.secondary-action` 复用 `.icon-button` 的可见焦点环(键盘焦点一致);`.rack-tools` 断点与控件区对齐(避免 560/820 错位)+ 极窄屏不与控件重叠;`.timeout-toast` 改 `white-space` 允许换行或限宽,极窄屏不截断;聊天留白槽延展 felt 背景去缝。
- **涉及:** `board.css`、`chat.css`(必要时)。
- **测试:** CSS 源断言 + 部署后真机(jsdom 测不了像素)。

### WS-G tap-to-place 键盘路径

- **改动:** 给 tap-to-place(`onCellTap`)补键盘可达:聚焦一张手牌后,用方向键/回车把它放到当前选中的目标格(复用既有 `resolveDropSlot`/分流),无障碍可操作。沿用既有键盘 Undo/Redo 的 hook 风格与 editable-target 守卫。
- **涉及:** `Board.jsx`/相关 hook、`Tile.jsx`/`GridSlot.jsx`。
- **边界:** 不与现有快捷键冲突;只在你的回合可用。
- **测试:** RTL——键盘把选中手牌放到目标格 → 调对应 move;非你回合不动。

### WS-H 首次开启 Hints tooltip

- **改动:** UX-pass T4 跳过的「首次开启 💡 Hints 的一次性 tooltip」补上:键 `rummycube:hintsTipSeen`,首次把 hints 由关→开时弹一句非阻塞说明 `These highlight tiles you can add to a group already on the table. You still need your 30-point opening meld first.`,记住后不再弹。
- **涉及:** `Board.jsx`/`HintsToggle.jsx`、`board.css`。
- **测试:** RTL——首次开启显示 tooltip + 写 flag;带 flag 不再显示。

## 4. 必须保持的不变量

- **服务器权威/反作弊不退化:** retrieveJoker / moveTiles / insertTilesWithPush 仍仅当前玩家、当前回合;非法 → `INVALID_MOVE`(immer 丢弃);客户端不写 `tilePositions`。
- **合法性裁定时机不变:** joker 换、挤位、摆放是几何/规则操作,run/set 合法性仍 `isBoardValid`/提交时裁定。
- **门控/无障碍:** 新动画 `@media (prefers-reduced-motion: no-preference)` / `fx.reduced()`(`window.matchMedia &&` 守卫);焦点环/键盘路径提升可达性。
- **应用内文案英文**;无新依赖;`node src/server.js` 可启动(服务端路径显式 `.js`);构建产物无新增 `console.log`;`timePerTurn` 毫秒。
- **既有能力不回归:** 长按整组、挤位、tap-to-place、键盘 Undo/Redo、断线/同步、超时公告、Hints 开关、主/次按钮、Exit、give-up 两次点击全部保留。
