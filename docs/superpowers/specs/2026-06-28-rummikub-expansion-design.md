## 规格 · 主选:公共合约 / Bounties
**钩子**:桌面变成一排悬赏合约的竞技场——谁先用桌上的牌摆出合约形态,谁拿分。

**组件**:合约卡 ×40–50(图标化、语言无关) + 计分垫 +(可选)沙漏。

**合约示例**:彩虹大组(4 色组)5 分 / 长龙(某色 6+ 顺)8 分 / 孪生(两个相同数字的组)6 分 / 纯净顺子(不含 joker 的 5 顺)5 分 / 调色板(同色出现在 3 个顺子)7 分 / 阶梯(三个递增长度的顺子)9 分。

**规则增量**:
1. 桌边翻开 3 张合约成"赏金行"。
2. 照基础出牌(起手 ≥30 不变);桌面合法后**可认领**此刻已满足的一张合约,但**本回合必须亲手放下/移动了至少一张参与该合约的牌**(防白嫖)。
3. 认领后补满至 3 张;每回合最多领 1 张。
4. 计分改积分制:已认领合约分 − 残牌分(joker −30),最高者胜或 100 分多局制;清空手牌仍结束本局但不再自动等于胜利。

**起手/胜负**:起手 ≥30 不变;由"先空手"改为积分制。

**人数/时长**:2–4 人,+10~20 分钟。

**设计论点**:把共享桌面变成被争夺的目标空间,深化核心引擎而非另起炉灶;带来"狙击/读人"互动与"快走 vs 刷分"取舍。

**Top 3 风险**:① AP/卡顿(限领 1 张、合约明置、上沙漏);② 狙击挫败/造王(贡献规则、明置信息、分值不致一锤定音、末局减少破坏);③ 判定争议(强图标、显式 joker 规则、速查卡)。

**首测问题**:限领数是否随人数缩放?贡献规则能否杜绝白嫖又不琐碎?合约分值带宽?已领明置还是暗扣?用 joker 的合约是否降分?

## 规格 · 备选:秘密任务 / Secret Missions
**钩子**:每人一张只有自己知道的目标,悄悄做出来加分。

**组件**:任务卡 ×40–50(图标化)+(可选)计分垫。

**任务示例**:亲手 4 色大组 +6 / 回收 joker 再打出 +5 / 一回合打 ≥10 张 +8 / 局末己方有 ≥3 个不同色顺子 +5 / 一张牌经重组同时延长两个集合 +7。

**规则增量**:每人抽 2 留 1;达成即翻面锁分(或局末揭示);最简版纯加分,基础胜负不变。

**人数/时长**:2–4 人,+5~10 分钟。

**设计论点**:近零摩擦的私人分歧目标,最稳的耐玩度提升。

**Top 3 风险**:① 运气依赖(围绕可塑造的模式设计、抽 2 留 1);② 任务失衡(数值/难度分级);③ 偏"多人单机"(加入需借用对手桌面牌组的任务)。

主推公共合约 + 任务卡作保险;首测盯死回合时长(AP),无法驯服则按 kill-switch 降级。**——此方向后被 owner 否决,转向网游互动重做。**
