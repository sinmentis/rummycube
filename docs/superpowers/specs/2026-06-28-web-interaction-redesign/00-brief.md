# 专家团队简报 · RummyCube 网游化互动重做

> 这是发给评审团每位专家的共享简报。各专家报告见同目录 `01`–`05`,综合结论见 `FINAL.md`。

## 产品
**RummyCube** —— 一个**线上 Web 多人 Rummikub 游戏**(React 18 + Vite + @dnd-kit;boardgame.io 0.50 服务器权威;Cloudflare Tunnel 后的单容器;个人维护),线上地址 game.shunlyu.com。**不是实体桌游**。已具备:2–4 人实时在线对局、拖放 + 点选、聊天气泡、连击打击感、自动整理桌面、移动端适配。基础规则:106 张牌(1–13 四色 ×2 + 2 joker)、顺子/组、起手 ≥30、之后自由重组共享桌面、先清空手牌者胜。

## Owner 本轮指令
- **否决**上一轮"目标/合约卡 + 积分制"方案。**不要积分制玩法。**
- 想要**更直接的玩家互动**(真正的 PvP 张力与参与感)。
- 按**网游思路**设计,**因为我们是 Web 界面,不是传统桌游**。彻底放开实体/成本约束。数字能力随便用:隐藏的同时状态、计时器/快棋钟、服务器校验的反应窗口、实时/半实时、持久档案、匹配、异步、服务器推送效果、垃圾牌/攻击系统等。
- **唯一要守的**:别破坏核心优雅(共享桌面重组这个解谜=灵魂)。经典模式保留可玩。这是叠加在现有 App 上的**新互动优先模式/重做**。

## 起始方向(供评估/取舍/组合,不限于此)
- **A. 垃圾牌攻击(竞技俄罗斯方块式)**:高效出牌把垃圾/惩罚牌推到对手手牌区;服务器推送的实时压力,直接 PvP。
- **B. 武器化共享桌面 + 反应窗口**:服务器开放短反应/挑战窗口;重组桌面可困住或释放对手的牌;靠干净出牌赚取的数字道具(冻结一组、偷看手牌、从池中偷牌)——Web 语境修好了实体"特殊牌"的体验问题(隐藏状态 + 计时器在屏幕上一目了然)。
- **C. 实时抢牌市场**:用实时共享池替代静态摸牌堆;服务器解决同时抢牌冲突。
- **D. 2v2 组队**:共享/联动手牌或共享目标,隐藏协作;Web 让通讯 + 隐藏队伍状态变得轻松。
- **E. 长线运营/持久层**:排位、异步随进随出房间、联赛、赛季(Web 原生留存元层,非核心互动)。
