> SUB-SKILL: subagent-driven-development。最高风险=转移打断,故 **spike 先行**。

## Tasks
### T1: 砸牌即时结算(无转移)+ 护盾抵消
playAbilityCard 接 `junk2/junk3/junk4`:`amount={2,3,4}`,有 target;若 `G.shields[target]` → 消耗护盾、无事;否则 target 摸 amount 普通牌(pushTilesToGrid)。立即生效(本回合 move)。**测**:三型摸 N、shield 抵、缺 target INVALID、非本人 INVALID、摸普通牌不摸能力卡。commit `feat(chaos): junk +N draw + shield block`。
### T2(spike): pendingJunk Stage 转移最小验证
注 `respondJunk` stage + move `transferJunk`/`acceptJunk`;A 砸→`G.pendingJunk={amount,target,from}` + setActivePlayers 给 target;target `acceptJunk`→摸 amount;超时默认 accept。**测**:Client 双人,A 砸 B→pendingJunk 置位、B accept 摸 N、链清。commit `feat(chaos): pendingJunk transfer stage (accept path + timeout default)`。
### T3: 转移叠加链
B 有砸牌→`transferJunk(card,C)`:amount+=cardN、target=C、链续;护盾在 accept 时抵整链;无上限。**测**:A+2→B,B+3→C,C accept 摸5;C 有 shield→0;链不封顶。commit `feat(chaos): junk chain stacking + shield clears chain`。
### T4: 接线 + playerView(pendingJunk 公开)+ 全量
wire stage/timeout，playerView 透 pendingJunk;DOM-free;集成测;gate(jest/build/lint/boot)。commit `feat(chaos): wire junk transfer + verify; SP2a complete`。
