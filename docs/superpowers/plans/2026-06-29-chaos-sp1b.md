**Goal:** 把 SP1a 落地的 chaos 后端骨架变成玩家**看得见、玩得动**的 UI:大厅 Classic/Chaos 开关、能力卡手区(自己的卡)、能力卡图鉴 FAB、以及**两张可玩卡**的出卡流(**Peek** 选目标→揭示其牌架;**Shield** 自挂),外加对手"有卡"小标 + Peek 施放光束(juice)。经典模式**完全不渲染任何 chaos UI**(gate 在 `G.mode==='chaos'`)。

## Global Constraints
- **经典零回归**:所有 chaos UI gate 在 `G.mode==='chaos'`;经典局**一个 chaos 元素都不渲染**、`createGame` 默认 `chaos=false`。既有前端测试全绿即证。
- **游戏内文案一律英文**(in-game UI English-only)。中文仅本设计文档。
- **保密**:UI **绝不显示对手能力卡的张数或内容**——对手只有"有卡"布尔小标(读 `G.abilityPresence`)。本人手牌读 `G.abilityHands[playerID]`。
- **复用既有**:FAB 仿 `ChatPanel` 的 `chat-fab`/`chat-root.open` 模式;卡面/稀有度用一套干净配色(白/蓝/金)+ 一个最小副提示(稀有度词),**不堆 W/B/G 字+点数+词**;tile 渲染复用 `Tile`/`TilePreview`;coach 卡复用现有观感。
- **无障碍**:光束/脉冲特效尊重 `prefers-reduced-motion`;稀有度不只靠颜色(配稀有度词)。
- **测试**:每个组件任务用 **RTL 行为测**(render + getByRole/getByText + fireEvent,仿 `src/tests/chat-fab.test.js`);布局(媒体查询 jsdom 测不了)用 **CSS-source 断言**(读 `.css` 文本断言规则存在,仿 `src/tests/board-visual-layout.test.js`)。
- 代码/标识符/注释/commit 英文;Conventional Commits + trailer `Co-authored-by: Copilot <223556219+Copilot@users.noreply.github.com>`。lint 不新增 error(基线 0 error / 91 warning)。

## 后端接口(SP1a 已就绪,本计划只消费)
- `G.mode` `'classic'|'chaos'`;`G.abilityHands[pid] = Card[]`,`Card={id,type,rarity}`;`G.abilityPresence[pid]=bool`(playerView 给,仅对手);`G.peekGrants[pid]=targetPid`;`G.shields[pid]=true`;`G.abilityDiscard`。
- playerView:当 `peekGrants[me]===t` 时,`G.tilePositions` 里会**包含** t 的 `gridId==='h'` 手牌 tile(否则被剥)。Peek 揭示面板就渲染这些 tile。
- move:`moves.playAbilityCard(cardId, target?)`——peek 需 `target`(目标 pid 字符串),shield 不需要;非本人回合/非 peek|shield/缺目标 → INVALID_MOVE(SP1a 已校验)。
- lobby:`createGame(playersCount, timePerTurn, chaos=false)`(SP1a 已加签名)。
- 10 种卡型(图鉴用):白 `peek,shield,junk2`;蓝 `junk3,force,wheel,bigwind`;金 `junk4,skip,lock`。SP1b **可玩仅 peek/shield**,其余在手区/图鉴里可见但点击不出(或灰显)。
